# Example Engagement — Engagement Spec

## Objective

Deliver a cloud-native data lakehouse on AWS for Example Client Ltd, replacing their legacy on-premises data warehouse and enabling real-time analytics across 15 priority data domains by Q4 2025.

## Context

Example Client Ltd is a mid-size financial services organisation undergoing digital transformation. Their current data infrastructure is over 10 years old and cannot support the real-time analytics required for their strategic roadmap. This engagement covers design and implementation of the replacement platform.

## Scope Summary

**In scope:**
- AWS lakehouse architecture design and implementation
- Migration of 15 confirmed priority data domains
- Data quality framework (Great Expectations)
- Self-service analytics layer (Power BI)
- Team training and knowledge transfer

**Out of scope:**
- Legacy system decommission
- Non-priority data domains
- Real-time streaming (Phase 2)

## Key Constraints

- Budget: £500,000 (T&M)
- Timeline: Q4 2025 go-live
- Technology: AWS only (client approved cloud provider)
- Resource: Client data team SMEs limited to 20% time commitment

## Success Criteria

1. Cloud lakehouse platform live and accepted by client by Q4 2025
2. 15 priority data domains migrated with zero data loss
3. Sub-5-minute data latency achieved for priority dashboards
4. 30% reduction in data infrastructure operating cost confirmed
5. Client data team self-sufficient on new platform post-training
