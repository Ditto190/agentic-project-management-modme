# RAID Management (Pre-Signature)

## Description

Identify, log, and manage Risks, Assumptions, Issues, and Dependencies prior to contract signature.

## Deliverables

- [[DLV-006]] RAID Register

## Key Questions

1. What could go wrong before we even sign the contract?
2. Which risks should be mitigated in the SoW terms?
3. Are there dependencies on third parties that need to be contractual obligations?

## Common Pitfalls

- Not starting the RAID register until after contract signature
- Missing dependencies that cause early delivery delays
