# Scope Definition & Baselining

## Description

Define, document, and baseline the project scope to serve as the reference point for change control and delivery planning.

## Responsibilities

- Facilitate scope workshops with client stakeholders
- Document in-scope and out-of-scope items
- Establish scope baseline and change control trigger
- Obtain formal sign-off on scope definition

## Deliverables

- [[DLV-002]] Scope Definition Document
- [[DLV-003]] Project Scope Baseline

## Applied In Scope

- SoW, MSA, Governance, Delivery Planning

## Key Questions

1. What does the client want us to deliver?
2. What are they explicitly NOT paying us to do?
3. What are the interfaces between our scope and the client's responsibilities?
4. What assumptions underpin this scope (log them all)?
5. What would constitute a scope change requiring a CR?

## Common Pitfalls

- Verbal scope agreements that aren't documented
- Vague deliverable descriptions that allow scope creep
- Missing out-of-scope list
- Not getting written sign-off from the right stakeholder
