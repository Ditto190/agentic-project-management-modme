# Scope Management Pattern

## Core Principle

> "Scope Management begins during the opportunity phase. By the time of contract signing, the contract should fully align with the solution plan, financials, and importantly, client expectations."

## Pattern Flow

```
Opportunity → Scope Workshops → Scope Definition (DLV-002)
→ WBS + Estimation → SoW (DLV-005) → Baseline Signed → CHANGE CONTROL ACTIVE
→ Any change → Change Request → Impact Assessment → Approval → Baseline Updated
```

## Anti-Patterns

| Anti-Pattern | Risk | Mitigation |
|-------------|------|-----------|
| Verbal scope agreement | No baseline for change control | Always document and sign off |
| No out-of-scope list | Client assumes everything included | Explicitly list out-of-scope items |
| First CR after delivery | Client sees change control as hostile | Introduce process at kick-off |
